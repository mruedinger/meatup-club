# Lessons Learned

- 2026-02-21: Initialized lessons log for remediation tracking.
- 2026-02-21: Route-action DB mocks must support both `prepare().first/run/all` and `prepare().bind().first/run/all`; otherwise tests can miss real query shapes and fail with mock-only type errors.
- 2026-02-23: When introducing stricter shared DB types, update dependent tests and typed query result casts in the same change; otherwise typecheck breaks on mock shape mismatches and `unknown` row types.
- 2026-02-23: Route-manifest integrity tests should use simple regex literals and validated filename normalization (`/\.tsx?$/`), because over-escaped patterns can silently invalidate the test harness.
- 2026-03-06: Notification helpers that aggregate external-send errors should redact recipient identifiers and return normalized `Error.message` strings; otherwise coverage runs drift between security expectations and runtime behavior.
- 2026-03-06: Debounced React component tests using fake timers need timer advancement wrapped in `act` (for example `await act(async () => vi.advanceTimersByTimeAsync(...))`), or state updates may never commit and the test will hang or assert stale UI.
- 2026-03-06: React Router route-module component tests should cast the assembled props object once at the render boundary when generated route prop types are stricter than the exercised scenario; otherwise `npm run typecheck` fails on test-only missing route metadata even though the runtime behavior is correct.
- 2026-03-06: Route tests that hand work to Cloudflare `waitUntil()` need to capture and await the queued promise before asserting background side effects, or happy-dom teardown can abort the work and make the test look like the side effect never happened.
- 2026-03-06: GitHub Actions only executes workflows from the repository-root `.github/workflows/` directory; nested `app/.github/workflows/` files do not run and can create a false sense that CI is enforcing checks when it is not.
- 2026-03-06: Failure mode: adding a workflow test that imports `node:sqlite` under the default `happy-dom` Vitest environment; detection signal: CI fails during Vite import analysis with "Cannot bundle built-in module `node:sqlite`"; prevention rule: mark Node-only workflow suites with `// @vitest-environment node` so Vitest runs them in the server environment.
- 2026-03-06: Failure mode: landing tests that depend on newer Node built-ins while CI is pinned to an older Node major; detection signal: local tests pass on a newer machine but GitHub Actions fails with "No such built-in module"; prevention rule: whenever tests or tooling rely on a Node built-in like `node:sqlite`, align the CI `setup-node` version with the minimum supported runtime in the repo.
- 2026-03-06: In Vitest auth/session tests, constructed `Request` objects do not reliably expose `Cookie` headers and redirect `Response` objects do not expose `Set-Cookie`; stub `request.headers.get("Cookie")` directly and assert session helper mocks instead of the response header.
- 2026-03-07: `crypto.randomUUID()` is typed as a UUID template literal in TypeScript, so test stubs must return a real UUID-shaped string or `npm run typecheck` will fail even though the runtime mock would work.
- 2026-03-07: Some route forms render visible labels without `htmlFor` or nested control associations, so route tests should query those controls by stable `name` selectors unless the task also includes an accessibility markup fix.
- 2026-03-07: In this Vitest `happy-dom` setup, `localStorage` is not always a full `Storage` implementation inside route-component tests, so tests that depend on visit-state keys should stub `localStorage` explicitly instead of assuming `clear()` and `removeItem()` exist.
- 2026-03-07: Test helper defaults can accidentally mask “missing configuration” branches; when exercising env-var guards, pass an explicit falsey runtime value like `""` or use a sentinel-based helper API instead of relying on `undefined` overrides that collapse back to defaults.
- 2026-03-07: After a PR merges, stop using that branch for new work. Detection signal: `origin/main` contains the merge commit but the working branch keeps accumulating follow-up commits. Prevention rule: branch from the latest `origin/main` for each new slice and open a new PR instead of pushing more commits onto a merged PR branch.
